# AI协作宪法

本目录用于存放 AI 护肤助理项目的核心架构与协作规范文档，作为项目开发和后续演进的统一参考。

**协作主体**：本项目 **唯一真人** 为 **Owner**（你本人）；**PMO、UX、FE、BE 等** 均为 **Cursor 多会话 Agent**，职责见 **`team-roles.md`** 与 **`Agent/README.md`**。

---

## 产品与 AI 行为宪章（必读）

**[`宪章-AI护肤助理.md`](./宪章-AI护肤助理.md)**（版本 1.0）为产品与内容治理母版：角色与伦理、服务边界、表达规范、质检与问责、对外告知等。**系统提示词、拦截规则、产品话术与关键用户文案须与本宪章及附录对齐**；技术实现细节仍以本目录下架构 / API 文档为准。

## 使用建议

- 涉及 **AI 对话、报告话术、营销表达、风险与就医提示** 时，先对照 **`宪章-AI护肤助理.md`**
- 开发前先统一阅读 `architecture.md`
- 接口开发优先参考 `api.md`
- 数据建模优先参考 `database.md`
- 产品流程与状态流转优先参考 `workflow.md`

---

## 约定

- 架构以“快速落地 + 可扩展”为原则
- 前端统一访问 Go 服务
- Python 只承接 AI 能力与算法能力
- AI 输出尽量结构化
- 所有核心设计优先考虑后续 B 端算法 API 扩展
