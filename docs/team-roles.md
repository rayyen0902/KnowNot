# 岗位代号与协作架构（一人公司 + 多会话）

> 用途：你向不同 Cursor 会话或合作方分派任务时，**只喊代号**，职责不串线。  
> 原则：**哨兵**拍板价值观与商业；**参谋**拆方案与下指令；其余为执行岗。

---

## 一、架构总览（建议：**序匠** 与 **面匠** 分会话；巡检可参谋兼）

```text
                    ┌─────────┐
                    │  哨兵   │  你：价值观 / 合规 / 商业拍板
                    └────┬────┘
                         │
                    ┌────▼────┐
                    │  参谋   │  总顾问会话：对齐文档与宪章、里程碑、粘贴指令、验收标准
                    └────┬────┘
         ┌──────┬──────┼──────┬──────┬──────┬──────┐
         ▼      ▼      ▼      ▼      ▼      ▼      ▼
       序匠    面匠    网工    智匠    云工    连调    巡检
 交互专岗 UI落地 Go+库 Python 部署  联调  上线抽检
```

- **序匠（交互专岗）**：用户动线、页面跳转、加载/空态/错误态、轮询与超时、手势与禁用态等 **先在这里定稿**，再交给面匠只做 **UI 与代码落地、修补样式**；避免「面匠会话又长又兼交互」导致反复出 bug。  
- **面匠**：以 **序匠交付的交互说明**（或参谋代序匠写的短 spec）为唯一依据做组件与样式；**不再默认包办**复杂交互设计。  
- **巡检**：仍可参谋兼；独立会话时只做抽检，不写日常功能。

## 二、代号一览表（分派用）

| 代号 | 典型承载 | 职责边界（做什么 / 不做什么） |
|------|-----------|-------------------------------|
| **哨兵** | 你本人 | **做**：价值观与宪章底线、对外承诺口径、商业取舍、平台授权。**不做**：写代码、改库表。 |
| **参谋** | 本顾问会话 | **做**：读 `docs/` 与《宪章》、定优先级、拆里程碑、写「可复制粘贴」实施命令、验收清单、冲突升级到你。**不做**：替你点平台后台、替你保管密钥。 |
| **序匠** | Cursor「交互」专会话（**口头可称「交互岗」**） | **做**：用户路径与信息架构、关键状态机（含轮询/超时/失败分支）、页面间跳转与参数约定、动效与禁用规则；输出给面匠的 **一页交互说明**（步骤列表 + 状态表即可）。**不做**：视觉稿细调、大量组件代码（交面匠）；不擅自改医学/合规话术（须对照宪章）。 |
| **面匠** | Cursor「前端/UI」会话 | **做**：按 **序匠**（或参谋代写的交互 spec）做 Taro/React **UI 与实现**、组件与样式、与接口字段对齐的展示；交互有歧义时 **先问序匠或参谋**，再改代码。**不做**：从零包办复杂交互方案（应交序匠）；Go/Python 核心；擅自改价值观话术。 |
| **网工** | Cursor「Go + Supabase」会话 | **做**：对外 API、鉴权、会话与报告任务、落库、编排调 Python、错误码与 trace。**不做**：大段 Prompt 工程（交给智匠）。 |
| **智匠** | Cursor「Python + AI」会话 | **做**：Qwen 调用、Prompt 编排、结构化输出、图像/OCR、与宪章对齐的安全策略。**不做**：对外暴露 API（经网工）。 |
| **云工** | Cursor「运维/FC」会话或运维同学 | **做**：FC/轻量服务器、域名 HTTPS、环境变量、发布脚本、日志与告警入口。**不做**：产品文案与医学边界裁决（交哨兵/参谋）。 |
| **连调** | Cursor「联调」会话 | **做**：逐接口 cURL/Apifox、字段对齐表、Go↔Python↔前端三方对照、失败分支复现。**不做**：替你决策产品方向。 |
| **巡检** | 参谋兼岗，或独立会话 | **做**：上线前宪章敏感点抽检、关键路径回归清单、隐私与告知一致性。**不做**：日常功能开发。 |

## 三、任务分派口令（复制即用）

- `【哨兵】请确认：……（A/B，价值观或商业）`  
- `【参谋】请输出：下一阶段目标 + 给序匠/面匠/网工的粘贴指令 + 验收标准`  
- `【序匠】请给出：…… 的交互说明（状态表 + 跳转 + 边界）供面匠实现`  
- `【面匠】按序匠交互说明 + implementation-v1 / 页面映射表执行：……（以 UI 与修补为主）`  
- `【网工】按 api.md + integration-v1 执行：……`  
- `【智匠】按 architecture 第六章 + 宪章执行：……`  
- `【云工】按 infra 约定执行：……`  
- `【连调】按 integration-v1 与 cURL 清单执行：……`  
- `【巡检】按宪章 + implementation-v1 做上线前抽检：……`

## 五、你问的那位「刚才干活出 cURL 的」叫什么？

**叫「连调」。**  
以后凡接口字段、cURL、三方对齐、联调记录，都归 **【连调】** 会话。
