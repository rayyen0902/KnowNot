- **P-Home-Chat**：首页 = 聊天主场景  
- **P-Report-Shell**：报告壳（脱敏、未登录可进）

- **P-Auth**：登录 / 绑定

## 1. 待 FE 实现清单

1. **首页为聊天主场景**：`report/task/create` 成功后 **回到 P-Home-Chat**，不在问卷页钉死 loading。  
2. **`done` 后**：**不**自动跳转报告详情页；在 **会话流** 插入 **持久报告卡片**（未登录仅 **中性字段**，不透出报告实质内容，对齐定稿 #8）。  
3. **`pending` / `running`**：底部「护肤报告」Tab **置灰**；**强触**（建议阈值见 §3.2）在聊天中插入系统/AI 消息 **「报告加速生成中……」**，并做 **防抖**。  
4. **未登录点 Tab**：**P-Report-Shell（脱敏）** → **P-Auth**；**已登录点 Tab**：**P-Report-Detail**（已有报告时 **直达最后一次** 生成结果，对齐定稿 #2）。  
5. **Shell**：**不渲染** `report/detail` 实质字段；**登录成功后再** 展示完整报告（或进入 P-Report-Detail 后拉取展示）。  
6. **Auth 取消 / 返回**：回到 **P-Home-Chat** + **一条** AI **软提醒**（非强迫，对齐定稿 #9）。  
7. **问卷再入**：**「接着填 / 重新填」** 弹窗；报告详情 **「再次生成」** 二选一（修改问卷 / 直接重生成）及 **回聊等待** 叙事（对齐定稿 #3、#10、#4）。  
8. **轮询与失败态**：**2s / 60s** 与 `failed` 呈现须对齐 **[implementation-v1.md §5.2](../implementation-v1.md)** 与 **[定稿 §3.2「异步生成与等待」](./2026-05-03-ux-v0-decisions.md)**：**柔和、可重试、不恐吓**；Tab 禁用须 **无障碍可读**（说明原因或 `aria` 等价语义，宿主能力允许范围内）。

## 2. 强触阈值与防抖（§3.2 · FE 默认起点）

> 具体数值由 PMO 工单定稿；实现前可用下列 **起点** 开发，联调再调。

| 项 | 建议起点 |
|----|-----------|
| 强触判定 | 置灰 Tab 在 **约 1.5s 内** ≥ **3 次** 有效点击（或宿主等价事件），视为强触 |
| 防抖 | 同一会话内 **「加速生成」类聊天消息** 至多每 **≥8s** 插入一条，避免刷屏 |
