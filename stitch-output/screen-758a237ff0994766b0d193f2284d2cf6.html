# 页面交付清单（版本：2023-10-27） ## 1. 页面清单与路由关系 | 页面名称 | 路由路径 | 功能描述 | | :--- | :--- | :--- | | **订单列表** | `/pages/order/list` | 展示用户历史订单，支持状态筛选。 | | **订单详情** | `/pages/order/detail` | 展示特定订单的物流、商品明细及支付信息。 | | **商品详情** | `/pages/product/detail` | 展示护肤品详细信息、AI 匹配度及评价。 | | **下单页** | `/pages/order/create` | 确认购买商品、选择规格并提交订单。 | ### 路由链路 - **订单流**：订单列表 -> 订单详情 - **购买流**：商品详情 -> 下单页 -> 订单详情 --- ## 2. 页面级交互说明 ### 页面：订单列表 - **跳转按钮**： - “查看详情” -> `/pages/order/detail?id={orderId}` - **交互弹层**： - “取消订单”二次确认弹层（确认/取消按钮）。 - **分页与筛选**： - **顶部筛选**：固定在顶部的状态切换（全部/待付款/待发货/已完成）。 - **触底加载**：支持列表触底分页加载（page + pageSize）。 - **数据加载**： - 首屏自动触发列表请求。 - 支持下拉刷新（Pull to Refresh）重新请求第一页数据。 ### 页面：商品详情 - **跳转按钮**： - “立即购买” -> `/pages/order/create?skuId={skuId}` - **交互弹层**： - **规格选择**：点击购买后唤起规格选择弹层（支持颜色、容量等维度选择）。 - **评价系统**： - 评价列表支持独立分页加载。 - **异步加载**： - 商品基础信息（标题、价格、主图）与推荐商品列表异步分批加载。 --- ## 3. 核心硬约束（必须执行） 为确保品牌一致性与数据模型准确，以下内容严禁修改： - **核心文案**： - 必须使用：**“立即购买”**、**“提交订单”**、**“取消订单”**。 - **禁止**使用同义词（如“马上购买”、“确认支付”等）。 - **视觉顺序**： - 商品详情页必须遵循：`商品图 -> 标题价格 -> 规格选择 -> 详情介绍 -> 推荐商品`。 - **技术字段名**： - 接口与代码逻辑必须匹配：`orderId`, `skuId`, `productName`, `finalPrice`。 --- ## 4. 优化与调整项（软约束） - **图标适配**：允许将 UI 中的图标样式适配为抖音小程序原生组件库。 - **间距微调**：允许在保持视觉平衡的前提下，对元素间距进行 ±4rpx 的微调。 --- **[交付状态]：待开发对接** *说明：本清单关联的 UI 屏幕详见画布【交付专用】系列。*
